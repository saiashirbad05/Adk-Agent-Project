{

  "summary": "...",
  "status": "success"
}

Important naming rule:
- Public project/service name: `adk-agent`
- Internal ADK agent identifier: `adk_agent`

Create `main.py`

import os
import uuid
from contextlib import asynccontextmanager
from typing import Literal

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field

from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai.types import Content, Part

from adk_agent.agent import root_agent


APP_NAME = "adk_agent"
USER_ID = "cloud-run-user"


class SummarizeRequest(BaseModel):
    text: str = Field(..., min_length=1, description="Text to summarize.")


class SummarizeResponse(BaseModel):
    original_text: str
    summary: str
    status: Literal["success"]


@asynccontextmanager
async def lifespan(app: FastAPI):
    session_service = InMemorySessionService()
    runner = Runner(
        agent=root_agent,
        app_name=APP_NAME,
        session_service=session_service,
    )
    app.state.session_service = session_service
    app.state.runner = runner
    yield


app = FastAPI(
    title="adk-agent",
    version="1.0.0",
    description="Production-ready text summarization agent using Google ADK and Gemini.",
    lifespan=lifespan,
)


@app.get("/")
async def root():
    return {
        "service": "adk-agent",
        "status": "ok",
        "message": "Use POST /summarize to summarize text."
    }


@app.get("/healthz")
async def healthz():
    return {"status": "ok"}


@app.post("/summarize", response_model=SummarizeResponse)
async def summarize(payload: SummarizeRequest):
    try:
        text = payload.text.strip()
        if not text:
            raise HTTPException(status_code=400, detail="The 'text' field must not be empty.")

        session_id = str(uuid.uuid4())
        session_service = app.state.session_service
        runner = app.state.runner

        await session_service.create_session(
            app_name=APP_NAME,
            user_id=USER_ID,
            session_id=session_id,
        )

        message = Content(role="user", parts=[Part(text=text)])
        summary = ""

        async for event in runner.run_async(
            user_id=USER_ID,
            session_id=session_id,
            new_message=message,
        ):
            if event.is_final_response() and event.content and event.content.parts:
                parts = [part.text for part in event.content.parts if getattr(part, "text", None)]
                summary = " ".join(parts).strip()

        if not summary:
            raise HTTPException(status_code=502, detail="The agent returned an empty summary.")

        return SummarizeResponse(
            original_text=text,
            summary=summary,
            status="success",
        )
    except HTTPException:
        raise
    except Exception as e:
        print("ERROR IN /summarize:", repr(e))
        raise HTTPException(status_code=500, detail=str(e))


if __name__ == "__main__":
    import uvicorn

    port = int(os.environ.get("PORT", "8080"))
    uvicorn.run("main:app", host="0.0.0.0", port=port, reload=False)

SECTION 4: SETUP COMMANDS IN CLOUD SHELL

1. Set your project:

gcloud config set project YOUR_GCP_PROJECT_ID

2. Enable APIs:

gcloud services enable run.googleapis.com artifactregistry.googleapis.com cloudbuild.googleapis.com secretmanager.googleapis.com

3. Create project folder:

mkdir -p ~/adk-agent-project/adk_agent
cd ~/adk-agent-project

====================================================================

Create venv and install:

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install --no-cache-dir -r requirements.txt

Set Gemini key:

export GOOGLE_API_KEY="YOUR_REAL_GEMINI_API_KEY"

echo "${GOOGLE_API_KEY:0:8}"

Start local app:

uvicorn main:app --host 0.0.0.0 --port 8080 --log-level debug

Test local health:

curl http://127.0.0.1:8080/healthz

curl -X POST http://127.0.0.1:8080/summarize \
  -H "Content-Type: application/json" \

====================================================================
SECTION 6: ARTIFACT REGISTRY AND BUILD
====================================================================

Create repository:

gcloud artifacts repositories create adk-agent-repo \
  --repository-format=docker \
  --location=us-central1 \
  --description="Docker repository for adk-agent"

Important:
- If repository already exists, this command may fail.
- That is okay. Just continue.

Create image tag:

export PROJECT_ID="$(gcloud config get-value project)"
export IMAGE_URI="us-central1-docker.pkg.dev/${PROJECT_ID}/adk-agent-repo/adk-agent:$(date +%Y%m%d-%H%M%S)"
echo "$IMAGE_URI"

Build and push:

gcloud builds submit --tag "$IMAGE_URI"

Important:
- Use a fresh timestamp tag.
- Do not keep reusing `:latest` if deployment looks stale.

====================================================================
SECTION 7: SECRET MANAGER
====================================================================

Create or update secret:

echo -n "$GOOGLE_API_KEY" | gcloud secrets create GOOGLE_API_KEY --data-file=- || \
echo -n "$GOOGLE_API_KEY" | gcloud secrets versions add GOOGLE_API_KEY --data-file=-

Get project number:

gcloud projects describe "$(gcloud config get-value project)" --format="value(projectNumber)"

Grant Cloud Run access:

gcloud secrets add-iam-policy-binding GOOGLE_API_KEY \
  --member="serviceAccount:PROJECT_NUMBER-compute@developer.gserviceaccount.com" \
  --role="roles/secretmanager.secretAccessor"

Replace:
- PROJECT_NUMBER

====================================================================
SECTION 8: DEPLOY TO CLOUD RUN
====================================================================

If you previously set GOOGLE_API_KEY as a plain env var in Cloud Run, remove it first:

gcloud run services update adk-agent \
  --region us-central1 \
  --remove-env-vars GOOGLE_API_KEY

Then deploy:

gcloud run deploy adk-agent \
  --image "$IMAGE_URI" \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --port 8080 \
  --set-secrets GOOGLE_API_KEY=GOOGLE_API_KEY:latest

Get service URL:

gcloud run services describe adk-agent \
  --region us-central1 \
  --format="value(status.url)"

====================================================================
SECTION 10: REAL ERRORS FACED AND HOW TO FIX THEM
====================================================================

ERROR 1: No space left on device during pip install

Error:
- `Could not install packages due to an OSError: [Errno 28] No space left on device`

Fix:

rm -rf ~/.cache/pip
rm -rf ~/.cache
rm -rf ~/adk-agent-project/.venv
docker system prune -af || true

Then reinstall:

python3 -m venv .venv
source .venv/bin/activate
pip install --no-cache-dir -r requirements.txt


ERROR 2: Invalid ADK agent name

Error:
- `Found invalid agent name: adk-agent`

Cause:
- ADK internal `name` cannot contain hyphens.

Wrong:

name="adk-agent"

Correct:

name="adk_agent"


ERROR 3: GOOGLE_API_KEY not available in server tab

Symptom:
- local server starts
- summarize call returns 500
- `echo $GOOGLE_API_KEY` prints nothing

Fix:
- set the variable in the same tab where the server runs

Commands:

source .venv/bin/activate
export GOOGLE_API_KEY="YOUR_REAL_GEMINI_API_KEY"
uvicorn main:app --host 0.0.0.0 --port 8080


ERROR 4: Free-tier quota exhausted

Error:
- `429 RESOURCE_EXHAUSTED`
- `generate_content_free_tier_requests`

Cause:
- project/key had no usable free-tier quota

Fix options:
1. use another project/key with quota
2. wait for quota reset
3. use a model available to that key


ERROR 5: Model not found

Error:
- `models/gemini-1.5-flash is not found`

Cause:
- model name not supported by that key/API version

Fix:
- list models:

curl "https://generativelanguage.googleapis.com/v1beta/models?key=$GOOGLE_API_KEY"

- use an actually supported model

Final working model:

model="gemini-2.5-flash"


ERROR 6: Cloud Build image name invalid

Error:
- `invalid image name ... YOUR_GCP_PROJECT_ID ...`

Cause:
- placeholder not replaced

Wrong:

export IMAGE_URI="us-central1-docker.pkg.dev/YOUR_GCP_PROJECT_ID/adk-agent-repo/adk-agent:latest"

Correct:

export PROJECT_ID="$(gcloud config get-value project)"
export IMAGE_URI="us-central1-docker.pkg.dev/${PROJECT_ID}/adk-agent-repo/adk-agent:$(date +%Y%m%d-%H%M%S)"


ERROR 7: Cloud Run deploy env var type conflict

Error:
- `Cannot update environment variable [GOOGLE_API_KEY] ... already been set with a different type`

Cause:
- old deployment used plain env var
- new deployment used Secret Manager binding

Fix:

gcloud run services update adk-agent \
  --region us-central1 \
  --remove-env-vars GOOGLE_API_KEY

Then deploy again with:

--set-secrets GOOGLE_API_KEY=GOOGLE_API_KEY:latest


ERROR 8: Wrong URL tested

Symptom:
- got Google HTML 404 page
- route looked missing even though app was deployed

Cause:
- wrong URL or stale host was used

Fix:
- always fetch service URL directly:

gcloud run services describe adk-agent \
  --region us-central1 \
  --format="value(status.url)"

- then set:

export APP_URL="$(gcloud run services describe adk-agent --region us-central1 --format='value(status.url)')"


ERROR 9: Deployment looked stale

Symptom:
- code worked locally
- Cloud Run still served old behavior

Cause:
- image tag reuse confusion

Fix:
- always deploy a fresh timestamped tag

export IMAGE_URI="us-central1-docker.pkg.dev/${PROJECT_ID}/adk-agent-repo/adk-agent:$(date +%Y%m%d-%H%M%S)"


ERROR 10: `/healthz` confusion

What happened:
- `/summarize` worked on deployed service
- `/healthz` checks were confusing during URL testing

Submission advice:
- the assignment mainly requires a working public endpoint
- your most important proof is a successful `/summarize` response

====================================================================
SECTION 11: FAILED OR MISLEADING COMMANDS TO AVOID
====================================================================

Avoid this internal agent name:

name="adk-agent"

Avoid unsupported model guesses:

model="gemini-1.5-flash"

Avoid placeholder image URI:

export IMAGE_URI="us-central1-docker.pkg.dev/YOUR_GCP_PROJECT_ID/adk-agent-repo/adk-agent:latest"

Avoid relying on `python main.py` if logs are unclear:

python main.py

Prefer:

uvicorn main:app --host 0.0.0.0 --port 8080 --log-level debug

Avoid reusing old `:latest` images if deployment seems stale.

Avoid assuming `GOOGLE_API_KEY` is available in every Cloud Shell tab.
Each tab is a separate shell session.
